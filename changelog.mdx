## Updating Your Instance

Follow these steps to upgrade safely and minimize downtime.

<Callout kind="alert">
  Always back up your data before updating. Review migration notes for each version.
</Callout>

<Steps>
  <Step title="Backup Your Data" icon="database">
    Export critical reports and configurations.

````bash
curl -X POST https://api.example.com/v1/export \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"reports": ["growth", "decisions"]}'
````
  </Step>

  <Step title="Download Latest Release" icon="download">
    Fetch the new version using your preferred package manager.
  </Step>

  <CodeGroup tabs="npm,Docker">
````javascript
// npm
npm install alchemetryx-consultation@latest
````

````dockerfile
# Docker
docker pull alchemetryx/consultation:latest
docker run -d -p 3000:3000 alchemetryx/consultation:latest
````
  </CodeGroup>

  <Step title="Run Migrations" icon="settings">
    Execute database migrations.

````bash
npm run migrate
# or
docker exec container npm run migrate
````

    Review version-specific notes in the [release assets](https://github.com/alchemetryx/consultation/releases).
  </Step>

  <Step title="Verify Deployment" icon="check-circle">
    Test key endpoints and dashboards.
  </Step>
</Steps>

## Migration Notes

<Expandable title="v1.2.0 Migration Details" default-open="true">
  Update your API calls to use the new `{growthMetrics}` endpoint:

````javascript
// Old
const metrics = await fetch('https://api.example.com/v1/metrics');

// New
const metrics = await fetch('https://api.example.com/v1/growth-metrics', {
  headers: { Authorization: `Bearer ${YOUR_TOKEN}` }
});
````
</Expandable>

<Expandable title="Breaking Changes in v1.0.0">
  Migrate from API keys to JWT:

  <ParamField header="Authorization" param-type="string" required="true">
    Use `Bearer {token}` format.
  </ParamField>
</Expandable>
